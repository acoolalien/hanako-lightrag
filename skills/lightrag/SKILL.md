# LightRAG 知识库

本 Skill 编排插件工具：

| 工具 | 用途 |
|------|------|
| `lightrag_query` | 查询知识库，6 种模式 + 跨库（workspace 逗号分隔） |
| `lightrag_insert` | 索引文档（批量，自动去重） |
| `lightrag_status` | 查看已索引文档和服务健康状态 |
| `lightrag_ws_list` | 列出所有知识库 |
| `lightrag_ws_create` | 创建新知识库 |
| `lightrag_ws_delete` | 删除知识库（需 `confirm: true`） |

## 多知识库

每个知识库有独立的索引和知识图谱。默认有一个 `default` 知识库。

- `workspace` 参数：所有工具都支持 `workspace` 参数指定目标知识库
- 查看所有知识库：使用 `lightrag_ws_list`
- 创建新知识库：使用 `lightrag_ws_create`，会自动更新配置
- 删除知识库：使用 `lightrag_ws_delete`（不可恢复，需确认）
- 切换知识库：图谱页顶栏下拉框切换，或工具调用时指定

## 索引前确认（必读）

收到索引请求时，如果用户未明确说明，**先追问以下信息**再调用工具：

1. **文档类型**：设定集 / 小说正文 / 技术文档 / 对话记录 / 研究资料 / 其他
2. **主要用途**：写作参考 / 设定一致性验证 / 跨文档关系推理 / 事实检索 / 探索浏览
3. **是否需要独立知识库**：如果这批文档属于独立项目或作品，建议用 `lightrag_ws_create` 创建专用知识库，避免与默认库混用

追问示例：
> "这批文档是什么类型？设定集还是小说正文？如果是设定集，我建议用 `local` 模式查更准。另外需要为它建一个独立知识库吗？"

## 文档类型 → 索引策略

| 文档类型 | 推荐查询 mode | 分块建议 | 原因 |
|----------|:----------:|------|------|
| 设定集（世界观/角色/组织） | `local` | chunkSize 800~1200 | 实体密集，精确匹配实体名最关键 |
| 小说正文 | `global` | chunkSize 1500~2000 | 关系/情节/因果链跨章分布 |
| 技术文档 | `mix` 或 `naive` | 默认 1200 | 概念 + 代码片段混合 |
| 对话/访谈记录 | `local` | chunkSize 600~1000 | 短句多，实体分散，小分块更准 |
| 研究资料/论文 | `mix` | 默认 1200 | 需要综合语义 + 关系 |
| 混合型（不知道类型） | `mix` | 默认 1200 | 安全默认值 |

> 分块大小在插件设置中全局配置。如需为特定文档类型调整，在索引前提醒用户修改配置。

## 查询模式选择指南

| 用户意图 | 推荐 mode | 说明 |
|----------|----------|------|
| 查具体事实/设定/定义 | `local` | 实体聚焦，精确匹配 |
| 跨文档推理/关系追踪/情节梳理 | `global` | 知识图谱社区摘要，关系穿透力强 |
| 综合查询（不确定该用哪个） | `mix` | 图谱 + 向量 + 可选的 Rerank |
| 简单关键词搜索 | `naive` | 纯向量检索，最快 |
| 只要原文不要 AI 总结 | `only_need_context: true` | 返回检索片段，Agent 自己推理 |

## 索引指南

1. **首次使用**：先确认文档类型和用途 → 决定是否创建独立知识库 → 批量索引
2. **批量索引**：多个文件多次调用 `lightrag_insert`，每次传入单文件内容。`filePath` 参数用于溯源
3. **增量更新**：文件修改后再次 insert 同一文件，LightRAG 自动按内容 MD5 去重
4. **追踪进度**：每次 insert 返回 `track_id`，用 `lightrag_status` 查看处理状态
5. **知识库隔离**：不同项目的文档放入不同 workspace，避免跨作品概念污染

## 上下文预算规则

1. 查询默认 `topK: 5`，最大 50
2. 返回结果超过 8000 字符会被截断
3. 查询噪声大时先用 `only_need_context: true` 看原文再决定
